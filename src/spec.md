## Role-Based Authentication System

### Super Admin Role
- **Permanent Super Admin**: The project owner's Internet Identity is automatically assigned as the permanent Super Admin with unrestricted privileges
- **Automatic Assignment**: Super Admin role is initialized on first login and cannot be revoked or modified
- **Unrestricted Access**: Super Admin has full access to all administrative features and content management functions
- **User Management**: Super Admin can create, modify, and delete Admin User accounts
- **Role Assignment**: Super Admin can grant and revoke Admin User roles and permissions
- **System Override**: Super Admin bypasses all authorization checks and has complete system control
- **Analytics Access**: Super Admin has full access to visitor analytics and login tracking data

### Admin User Role
- **Role-Based Access**: Admin Users have restricted access to specific content management areas based on assigned permissions
- **Internet Identity Authentication**: Admin Users authenticate through Internet Identity system
- **Permission Boundaries**: Admin Users can only access features and content areas explicitly granted by Super Admin

- **Role Restrictions**: Admin Users cannot create other admin accounts or modify their own permissions

- **Consistent visual presentation**: All uploaded images—from device uploads or external sources (Google Drive, OneDrive, Dropbox, Google Photos)—scale appropriately without distortion or unwanted stretching on wide screens
- **Animated GIF preservation**: Animated GIFs maintain proper playback and animation while conforming to the banner dimensions and aspect ratio requirements

- **Role-Based Access**: Banner management restricted to users with appropriate permissions (Super Admin and authorized Admin Users)
- **Enhanced backend proxy image-fetching system**: Dedicated backend proxy endpoint using IC's `http_outcalls` module that securely retrieves external images from Google Drive, OneDrive, Dropbox, and Google Photos. The proxy features:
  - **Automatic URL detection and conversion**: Detects non-direct URLs and converts them to proper direct access URLs using multiple conversion strategies
  - **Robust error recovery**: Implements retry mechanisms, timeout handling, and fallback URL processing for maximum compatibility
  - **MIME type validation**: Verifies content type headers to ensure only valid image formats are processed
  - **Cache prevention**: Uses cache-busting headers and parameters to prevent stale content delivery
  - **Comprehensive error handling**: Returns specific error messages for private files, invalid URLs, network issues, and unsupported formats
  - **Blob data processing**: Fetches image data as a blob and returns it with correct `Content-Type: image/*` headers to resolve CORS and access permission issues
- **Advanced client-side link conversion system**: Frontend automatically detects and converts external sharing links from Google Drive, OneDrive, Dropbox, and Google Photos using an enhanced `urlConverter.ts` utility that:
  - **Automatic cloud provider detection**: Instantly identifies the cloud storage provider (Google Drive, OneDrive, Dropbox, Google Photos) from pasted sharing URLs
  - **Comprehensive Google Drive URL parsing**: Extracts file IDs from all Google Drive URL formats including `/file/d/FILEID/`, `/d/FILEID/`, `/open?id=FILEID`, `/uc?id=FILEID`, `/uc?export=view&id=FILEID`, shared view links, and edit links
  - **Google Photos URL parsing**: Detects and processes Google Photos shared links including album links, individual photo links, and direct photo URLs, extracting photo IDs and converting to direct image URLs
  - **Standardized direct URL generation**: Converts all detected Google Drive URLs to the standard direct-view format `https://drive.google.com/uc?export=view&id=FILEID` and Google Photos URLs to direct image access format
  - **OneDrive URL conversion**: Processes OneDrive shared URLs by extracting sharing tokens and converting to direct image URLs
  - **Dropbox URL conversion**: Handles Dropbox shared URLs by modifying query parameters to create direct download links
  - **Client-side validation and conversion**: All link detection, conversion, and initial validation occurs entirely on the frontend without backend dependencies
  - **Instant conversion feedback**: Provides immediate visual feedback when sharing links are detected and converted to direct URLs
  - **Backend proxy routing**: Routes converted URLs through the enhanced backend proxy endpoint for secure fetching and proper header handling
  - **Automatic fallback URL processing**: Backend proxy automatically attempts multiple URL conversion strategies for non-direct URLs to ensure successful media access
- **Enhanced live validation feedback system**: When external URLs are pasted, the system displays real-time validation status messages:

  - **Instant preview refresh**: Successfully validated URLs automatically update the admin panel preview ("All Uploaded Banners" section) and live banner above the navbar
- **Guaranteed backend proxy validation**: External image URLs are automatically validated through the enhanced backend proxy system before display, ensuring proper accessibility and format compatibility with comprehensive error recovery
- **Real-time banner updates with enhanced validation**: Successfully validated URLs through the backend proxy automatically update the live banner above the navbar in real time without page refresh, with comprehensive validation ensuring proper media access and instant fallback prevention
- **Instant conversion with enhanced backend proxy processing**: When external links are pasted or entered, the system immediately processes them client-side, validates through the enhanced backend proxy with improved error recovery, and displays the image both in the admin panel preview ("All Uploaded Banners" section) and live banner above the navbar
- **Real-time preview synchronization with validation**: Successfully validated URLs automatically update both admin preview and live banner display instantly with immediate re-rendering of image components and prevention of fallback image display
- **Automatic fallback replacement with validation**: Validated URLs replace default fallback images immediately after successful backend proxy validation with comprehensive error handling
- **Enhanced cache invalidation**: Banner components use timestamp-based cache-busting, version tracking, and force immediate UI state updates to ensure newly uploaded or proxy-validated images display instantly without fallback content
